# 红果搭建工具

> 红果短剧广告项目自动化搭建工具，适用于巨量创量平台，提供可视化 GUI 界面，支持多种投放模式批量搭建。

## 功能特性

### 核心搭建

- **四种投放模式**：安卓-每留 / 安卓-七留 / iOS-每留 / iOS-七留
- **激励视频搭建**：安卓-激励每留 / 安卓-激励七留
- **批量自动化**：一键完成从策略选择到审核提交的全流程
- **差异化搭建**：按账户分组 × 剧名组合，实现多维度差异化投放
- **断点续传**：支持中断后自动恢复未完成任务

### 素材管理

- **按 ID 精确搜索**：通过预设素材 ID 批量搜索并选取素材
- **按关键词筛选**：使用剧名等关键词智能筛选素材
- **素材推送**：将素材批量推送到指定投放账户
- **素材爬取**：自动爬取符合条件的素材 ID 列表
- **历史记录**：完整的素材使用历史追溯

### 监测链接

- **三类链接配置**：展示监测 / 点击监测 / 视频播放监测
- **按剧名映射**：每部剧独立配置对应的监测链接
- **批量导入**：支持从 Excel 等格式批量导入链接

### 推广链工具

- **推广链生成**：自动为多部剧批量生成推广链接
- **推广链分拆**：按平台和留存策略自动分类归档
- **激励链管理**：独立的激励推广链生成与分拆

### 辅助工具

- **RTA 批量设置**：快速配置 RTA 生效范围
- **RTA 状态检测**：批量检测 RTA 启用状态
- **每日任务管理**：创建和追踪每日搭建任务
- **搭建记录统计**：账户数、项目数等数据汇总

### 搭建流程（8 步）

1. 选择投放策略
2. 选择媒体账户（支持批量多账户）
3. 关联产品（按剧名搜索匹配）
4. 填写监测链接（展示/点击/播放）
5. 选择定向包（自动匹配账户）
6. 填写项目名称（自动生成规范命名）
7. 填写广告名称（动态编号）
8. 选择素材并提交审核

## 环境要求

- **操作系统**：Windows 10 / 11
- **Python**：3.10+
- **浏览器**：Chrome（需以调试端口模式启动）
- **Node.js**：16+（仅开发模式需要）

## 安装

### 方式一：直接运行（推荐）

下载 [Releases](https://github.com/q57wu/hongguo-build-tool/releases) 中的打包版本，双击 `红果搭建工具.exe` 即可使用。

### 方式二：源码运行

```bash
# 1. 克隆仓库
git clone https://github.com/q57wu/hongguo-build-tool.git
cd hongguo-build-tool

# 2. 安装 Python 依赖
pip install -r requirements.txt
pip install pywebview

# 3. 安装 Playwright 浏览器（可选，用于自带浏览器）
playwright install chromium

# 4. 安装前端依赖并构建
cd frontend
npm install
npm run build
cd ..

# 5. 启动
python app.py
```

## 使用说明

### 1. 启动 Chrome 调试模式

```bash
chrome.exe --remote-debugging-port=9222
```

或在工具的设置页面配置 Chrome 路径后，由工具自动启动。

### 2. 配置投放参数

在设置页面完成：
- 操作人名称
- 各模式的策略名、素材账户、定向包关键词
- 导入剧单及对应监测链接

### 3. 开始搭建

1. 选择投放模式（安卓-每留 / 安卓-七留 / iOS-每留 / iOS-七留）
2. 确认账户分组与剧名配置
3. 点击"开始搭建"
4. 实时查看日志与进度
5. 搭建完成后查看统计

### 数据格式参考

```
账户ID1
账户ID2

剧名

展示监测链接
点击监测链接
视频播放监测链接

素材ID1 素材ID2 素材ID3

===

（下一组，用 === 分隔）
```

## 开发

```bash
# 启动开发模式（前后端热更新）
# 终端 1：前端
cd frontend && npm run dev

# 终端 2：后端
set DEV=1 && python app.py
```

### 项目结构

```
├── app.py                 # 主入口（pywebview 窗口）
├── start.py               # 启动脚本
├── backend/               # 后端逻辑
│   ├── api.py             # JS Bridge API
│   ├── bridge.py          # 事件桥接
│   ├── build_engine.py    # 搭建引擎
│   ├── core/              # 核心业务（搭建步骤、推广链等）
│   ├── services/          # 服务层（浏览器、进度、任务）
│   ├── tools/             # RTA 等工具
│   └── utils/             # 工具函数
├── frontend/              # Vue 3 前端
│   ├── src/
│   │   ├── views/         # 页面组件
│   │   ├── stores/        # Pinia 状态管理
│   │   ├── services/      # API 调用层
│   │   └── composables/   # 组合式函数
│   └── vite.config.js
├── assets/                # 静态资源
└── requirements.txt       # Python 依赖
```
